# 🔍 前端代码走查 — 使用说明

> 一个 Kiro Skill，用 AI 对 React + Antd 5 前端代码进行自动化走查，输出走查报告。

---

## 📌 两种使用模式

### 模式一：独立走查

不需要需求文档，直接指定文件范围：

```
代码走查 src/view/organization/businessSwitch/**/*.tsx
```

### 模式二：需求关联走查

配合 daily-workflow 使用，从设计文档自动获取文件范围：

```
CR（在 daily-workflow 第五步自动触发）
```

---

## 📢 常用命令

| 说什么 | 做什么 |
|--------|--------|
| `代码走查` + 文件路径 | 独立走查指定文件 |
| `CR` + 文件路径 | 同上 |
| `code review` | 走查（会询问文件范围） |

---

## 📋 走查维度

### ✅ 基础维度（每次必检）

1. ✅ 代码规范 — 命名、函数风格、类型定义、导入、注释、格式、路径别名
2. 🛡️ 逻辑健壮性 — 空值保护、JSON.parse 保护、条件判断完整性、表单重置、错误处理
3. 🌐 接口调用 — 请求时机、loading、防重提交、竞态条件、参数类型、保存后刷新
4. ⚡ 性能与可维护性 — 冗余 state、组件/函数大小、嵌套深度、防抖、硬编码、console.log
5. 💡 优化建议(P3) — useState/useMemo/useCallback 滥用、useWatch 风险、状态提升、内联函数

### 🎯 可选维度

6. 🔗 与设计文档一致性 — 仅需求关联走查时检查
7. 🧩 专项检查 — 根据用户选择启用：
   - 🔄 Antd 升级专项（Table/Form/Modal/Select/DatePicker/Upload）
   - ⚛️ React 升级专项（17→18：自动批处理、Strict Mode、事件系统）
   - 🎁 IP 联名套餐专项
   - 📣 营销活动专项
   - ✏️ 自定义专项（用户描述场景，AI 生成检查项）

---

## 🏷️ 严重程度

| 级别 | 含义 | 处理方式 |
|------|------|----------|
| 🔴 P0 | 紧急重要，必须现在就修改 | 立即修复 |
| 🟠 P1 | 重要但不紧急，可以稍后改 | 排期修复 |
| 🟡 P2 | 不是很关键，可以以后再改 | 视情况修复 |
| 🟢 P3 | 优化建议，提升代码质量 | 有空再改 |

---

## 📊 输出

走查报告默认在对话中展示，也可以写入飞书文档。包含：
- 📊 走查概览（各维度通过/问题数统计）
- 📝 问题详情表格（文件、类型、描述、严重程度、修复建议、是否已修复）
- 💡 AI 建议和风险提示
