# AI Code Review 规范（daily-workflow 专用）

## 走查维度

走查维度和检查项统一使用 `cr-general` skill 的定义（`.kiro/skills/cr-general/SKILL.md`），包括：

1. 代码规范检查
2. 逻辑健壮性检查
3. 接口调用检查
4. 性能与可维护性检查
5. 交互与代码逻辑优化建议（P3）
6. 与设计文档一致性（daily-workflow 必检）
7. 专项检查（可选）

> 维护一份检查项，两个 skill 共用。cr-general 更新后 daily-workflow 自动生效。

## daily-workflow 特有规则

### 走查范围

从设计文档的"修改文件表格"自动获取，无需用户手动指定。

### 走查后操作

1. 代码规范类问题直接自动修复
2. 逻辑类问题列出并等用户确认后修复
3. 修复完成后自动更新飞书走查报告文档中对应问题的"是否已修复"状态

### 走查文档

在飞书 `Kiro/YYYYMMDD_[需求名]_需求分析/` 目录创建，标题格式：`[需求名]_AI走查报告`

### 严重程度

P0（紧急重要，必须现在就修改）/ P1（重要但不紧急，可以稍后改）/ P2（不是很关键，可以以后再改）/ P3（优化建议，提升代码质量和可维护性）

### 自我学习

走查中发现的新问题模式，优先更新到 `cr-general` skill 中，确保两边一致。
