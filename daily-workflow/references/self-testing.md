# 自测规范

## 一、自测清单生成规则

自测清单基于需求分析和设计文档自动生成，必须覆盖以下维度，按优先级排列。

---

## 二、用户输入与操作边界

### 表单输入

| 检查项 | 说明 |
|--------|------|
| 必填校验 | 所有 `required` 字段为空时提交，是否有正确提示 |
| 最大长度 | 输入超出 maxLength 限制，是否截断或提示 |
| 最小长度 | 输入不足 minLength，是否提示 |
| 特殊字符 | 输入 `<script>`、emoji、中文标点、空格等，是否正确处理 |
| 纯空格 | 输入全空格，是否被 trim 或拦截 |
| 数字边界 | 数字输入 0、负数、小数、超大数值，是否正确校验 |
| 日期边界 | 开始日期 > 结束日期、选择过去/未来日期，是否拦截 |
| 重复提交 | 快速连续点击提交按钮，是否有防重处理（loading/disabled） |

### 列表与分页

| 检查项 | 说明 |
|--------|------|
| 空数据 | 列表无数据时是否展示 Empty 组件 |
| 单条数据 | 只有一条数据时分页是否正常 |
| 大量数据 | 数据量大时是否有分页/虚拟滚动，页面是否卡顿 |
| 翻页后操作 | 翻页后编辑/删除，返回列表是否保持当前页 |
| 搜索+分页 | 搜索后是否重置到第一页 |

### 弹窗与交互

| 检查项 | 说明 |
|--------|------|
| 弹窗关闭 | 关闭弹窗后表单数据是否清空 |
| 确认操作 | 删除等危险操作是否有二次确认（Popconfirm/Modal.confirm） |
| 加载状态 | 异步操作是否有 loading 状态，防止重复操作 |
| 键盘操作 | Enter 是否触发提交，Esc 是否关闭弹窗 |

## 三、接口参数完整性与类型

### 请求参数

| 检查项 | 说明 |
|--------|------|
| 必传参数 | 所有必传参数是否都有传递，不能为 undefined/null |
| 参数类型 | number 类型参数是否传了 string（常见于表单取值） |
| 枚举值 | 枚举参数是否在合法范围内 |
| 分页参数 | page/pageSize 是否正确传递，首页是 0 还是 1 |
| 空数组 | 数组参数为空时传 `[]` 还是不传，与后端约定一致 |
| 日期格式 | dayjs 格式化后的字符串是否符合后端要求（如 `YYYY-MM-DD HH:mm:ss`） |

### 响应处理

| 检查项 | 说明 |
|--------|------|
| 空响应 | 接口返回空数据（null/undefined/空数组）时页面是否正常 |
| 字段缺失 | 响应中某个字段缺失时是否有兜底（可选链 `?.`） |
| 错误码处理 | 业务错误码是否正确处理并提示用户 |
| 网络异常 | 断网/超时时是否有友好提示 |
| 并发请求 | 快速切换页面/tab 时，旧请求响应是否被忽略（竞态处理） |

### 类型安全

| 检查项 | 说明 |
|--------|------|
| 接口类型定义 | 请求参数和响应是否都有 TypeScript 类型定义 |
| any 使用 | 是否有不必要的 `any`，能否用具体类型或 `unknown` 替代 |
| 类型断言 | `as` 断言是否合理，是否有更安全的类型收窄方式 |

## 四、ESLint 与代码规范检查

### 自动化检查（必须全部通过）

| 检查项 | 命令/方式 |
|--------|-----------|
| ESLint | `npx eslint src --ext .ts,.tsx` 无 error |
| Stylelint | `npx stylelint "src/**/*.less"` 无 error |
| TypeScript | 无编译错误（`getDiagnostics` 检查） |

### 人工复核

| 检查项 | 说明 |
|--------|------|
| console.log | 是否有遗留的 `console.log` 调试代码 |
| 注释清理 | 是否有遗留的注释代码块（被注释掉的旧代码） |
| TODO 标记 | 是否有未处理的 `// TODO` 标记（非预期保留的） |
| 硬编码 | 是否有硬编码的 URL、ID、文案等，应提取为常量 |
| 函数长度 | 单个函数是否超过 50 行 |
| 组件大小 | 单个组件文件是否超过 300 行 |
| 命名规范 | 变量/函数/组件命名是否符合 coding-standards 规范 |
| moment 引用 | 是否误引入了 moment（必须用 dayjs） |
| func-style | 是否有 function 声明（必须用箭头函数表达式） |

## 五、业务逻辑边界

| 检查项 | 说明 |
|--------|------|
| 权限控制 | 无权限时按钮/菜单是否隐藏或禁用 |
| 状态流转 | 业务状态变更后页面展示是否正确（如：待审核→已通过） |
| 数据联动 | 上级选择变更后，下级数据是否正确刷新/清空 |
| 默认值 | 新建/编辑表单的默认值是否正确 |
| 回显 | 编辑时数据是否正确回显（特别是下拉框、日期、富文本） |

---

## 六、浏览器兼容性

| 检查项 | 说明 |
|--------|------|
| Chrome 最新版 | 主要开发浏览器，必须完全正常 |
| Safari（如需支持） | CSS 兼容性（flex gap、sticky 等）、日期格式差异 |
| 移动端 WebView（如需支持） | 触摸事件、软键盘弹出、安全区域适配 |
| 缩放/分辨率 | 浏览器缩放 80%~150% 时布局是否正常 |

---

## 七、自测清单输出格式

在飞书文档中按以下表格格式输出：

| 序号 | 分类 | 测试项 | 测试步骤 | 预期结果 | 实际结果 | 是否通过 |
|------|------|--------|----------|----------|----------|----------|
| 1 | 用户输入 | 必填字段为空提交 | 清空所有必填字段，点击提交 | 显示必填提示 | | |

- "实际结果"和"是否通过"由用户手动填写
- 分类按本文档的维度划分：用户输入、接口参数、ESLint规范、业务逻辑、浏览器兼容
- 测试项从需求分析和设计文档中提取具体场景，不要写泛泛的通用项
