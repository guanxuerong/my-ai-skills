# 通用规则

## 文档存放

- 所有文档存放在飞书 `Kiro/` 根目录下对应子目录
- 文档中的链接必须是可点击的飞书链接

## 状态更新

- 每步完成后更新 daily 表格状态

## 日报更新

每次触发关键动作时，在 daily 表格当天日期列对应需求行中追加一条记录，用序号标识，格式如：

```
1.完成需求分析<br/>2.完成设计文档<br/>3.完成开发
```

- 每条记录使用"完成xxx"的表述，不要直接写关键词
- 每条记录之间使用 `<br/>` 换行（飞书表格单元格内换行必须用 `<br/>`，普通换行符无效）
- 用序号标识
- 同一天内多次触发则依次递增序号追加（不覆盖已有内容）
- 常见关键词：需求评审、需求分析、设计文档、开发、接口联调、自测、提测、跟进测试、Bug修复等
- 如当天日期列不存在，先在当前状态列后新增该日期列（列头格式 `MM/DD`）

## 决策原则

- **不确定的事情必须停下来问用户，不要猜**

## 上下文传递

- 每步生成的关键信息（需求名、分支名、文档链接、接口文档地址等）需在后续步骤中自动引用，不要重复询问用户
- 当获取到新的关键信息时（如用户提供接口文档地址），必须立即回填到之前已创建的相关文档中（如设计文档的元信息表格），不要等用户提醒
- 如果会话中断后恢复，先读取 daily_work 表格获取当前需求的状态和已有信息，从断点继续

## 多需求并行

- 同一时间可能有多个需求在不同阶段，每个需求独立跟踪
- 用户切换需求时，通过需求名或分支名定位到对应的 daily 表格行
- 日报列中同一天可以有多个需求的记录

## 中断与恢复

- 用户可以在任意步骤暂停，后续通过"继续 [需求名]"恢复
- 恢复时优先通过 `git branch --show-current` 获取当前分支名，匹配 daily_work 表格中的分支名列，自动定位到对应需求行
- 如果当前分支名能匹配到需求，则无需用户指定需求名，直接从断点继续
- 如果当前分支不匹配任何需求（如在 main/master 上），则读取 daily 表格的"当前状态"列，询问用户要继续哪个需求

## 自我学习与优化

在流程执行过程中，AI 应主动识别可优化的模式并提出改进建议。

### 触发条件

遇到以下情况时，主动向用户提出优化建议：

- 用户指出了流程中的问题或不合理之处（如"为什么没自动回填"）
- 同一类问题重复出现（如多次忘记同步更新文档）
- 用户提出了更好的做法（如"用 RangePicker 不要分开"）
- 流程执行中遇到了工具限制需要 workaround（如飞书 MCP 不支持电子表格）
- 发现规范文档中的描述与实际执行不一致

### 执行流程

1. **识别**：在执行流程中发现可优化点
2. **提出**：向用户描述问题和优化方案，说明影响哪些文件（skill.md / references/*.md / README.md）
3. **确认**：等待用户确认，不要自行修改
4. **执行**：用户确认后，自动更新所有相关文件（skill.md、对应的 references/*.md、README.md），确保三处保持一致
5. **记录**：如有飞书实践文档（如"AI 驱动前端开发工作流"），也同步更新相关章节

### 注意事项

- 不要积攒多个优化点一次性提出，发现一个就及时提出
- 优化建议要具体，说明"改什么"和"为什么改"
- 用户拒绝的优化建议不要重复提出

## 飞书文档写作注意事项

### 换行规则

- **表格单元格内换行**：必须使用 `<br/>`，普通换行符（`\n`）在表格单元格内无效，会被合并为一行
- **段落之间**：需要空行分隔，单个换行符会被飞书合并为同一段落
- **callout / grid / column 等组件内**：内容前后需要空行，否则内容可能无法正确渲染
- **lark-table 单元格内**：`<lark-td>` 标签内的内容前后必须空行

### 编辑操作注意

- **优先使用 `replace_range` + 精确定位**，避免 `replace_all` 误伤其他内容
- **表格类内容修改**：用 `selection_by_title` 按章节定位更安全，避免定位到表格内部结构
- **`replace_all` 慎用**：会替换文档中所有匹配项，容易误改不相关的内容（如把所有"待填写"都替换掉）
- **插入内容时**：`insert_before` / `insert_after` 的定位文本要足够长（10-20 字符），确保唯一匹配
- **飞书表格行级定位困难**：飞书云文档中的表格（lark-table 或 Markdown 表格）转换后结构复杂，无法精确定位到某一行。需要更新表格中的某几行时，用 `selection_by_title` 替换整个章节（包含表头和所有行），而不是尝试定位单行
