# 提测文档模板

## 标题格式

`MMDD【前端】[需求名]--提测`

> 如需求属于专项（如优化专项），可改为 `MMDD【前端优化专项】[需求名]--提测`，询问用户确认标题分类。

## 文档内容

| 字段 | 值 | 填写方式 |
|------|------|----------|
| 提测名称 | `[需求名]` | 自动填充 |
| 前端开发人 | `<mention-user id="ou_737b699e537129108cb65be2f4d9c7cd"/>` | 固定值，必须使用 mention-user 真实 @ |
| 测试人 | | 留空，由用户自行填写 |
| 设计文档 | `[设计文档飞书链接]` | 自动填充 |
| 飞书项目链接 | | 留空，由用户自行填写 |
| 应用名 | fe-teabydao | 固定值 |
| 分支名 | `[第一步生成的分支名]` | 自动填充 |
| 冒烟执行记录 | | 留空，由用户自行填写 |
| 合入分支 | test | 固定值 |
| 前置条件 | 接口上线 | 固定值 |
| 是否需联调 | `[是/否]` | 只要涉及接口调用就填"是"，仅纯前端改动（如样式调整）才填"否" |
| 接口文档 | `[接口文档链接]` | 自动填充（来自第三步或第四步） |
| 预估影响范围 | `[影响的模块/页面列表]` | 基于设计文档自动生成 |
| code review 记录地址 | | 留空，由用户自行填写 |
| 备注 | | 留空，由用户自行填写 |

## 填写规则

- **自动填充**：从流程上下文中自动获取，无需用户手动输入
- **固定值**：模板预设值，一般不需要修改
- **留空**：无法自动获取的字段，留空让用户自行填写，不要填"待补充"等占位文字
- **@ 用户**：必须使用 `<mention-user id="open_id"/>` 语法真实 @，不要写纯文本 @xxx
